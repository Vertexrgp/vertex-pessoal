A Vertex opera sob uma **arquitetura de marca monolítica**: a marca principal sustenta todos os produtos.

```
VERTEX COMPANY
(marca mãe — identidade corporativa)
        │
        ▼
   VERTEX OS
(produto principal — sistema operacional da vida)

        ├── Financeiro
        ├── Performance
        ├── Agenda
        ├── Viagens
        ├── Crescimento
```

### Como os módulos se relacionam à marca

Cada módulo do Vertex OS **não é uma submarca**. São funcionalidades de um sistema unificado. Eles devem usar:

- A mesma identidade visual do Vertex OS
- O mesmo sistema de cores e tipografia
- Nenhum logotipo próprio
- Diferenciação apenas por cor de acento (ver seção de cores)

## 4. IDENTIDADE DO PRODUTO — VERTEX OS

### Conceito

> **O sistema operacional da sua vida.**

O Vertex OS parte de uma analogia clara: assim como um sistema operacional organiza e dá acesso a todas as funcionalidades de um computador, o Vertex OS organiza as áreas mais importantes da vida de uma pessoa — finanças, saúde, aprendizado, agenda, metas, idiomas e viagens — em um único ambiente coeso.

### Experiência central

O produto deve transmitir três sensações em cada interação:

| Sensação | O que isso significa na prática |
|---|---|
| **Controle** | O usuário sempre sabe onde está e o que aconteceu |
| **Clareza** | A informação é exibida sem ruído ou ambiguidade |
| **Decisão** | O produto ajuda o usuário a tomar decisões melhores |

### Posicionamento

**Premium, simples e poderoso.**

- Premium não significa complicado. Significa feito com cuidado.
- Simples não significa sem funcionalidade. Significa fácil de usar.
- Poderoso não significa cheio de opções. Significa que cada função entrega o que promete.

### O usuário

Profissional ambicioso, 25–40 anos, que quer um sistema para organizar sua vida com o mesmo nível de qualidade que usa nas ferramentas do trabalho. Não quer planilhas. Não quer apps fragmentados. Quer um sistema único, bonito e confiável.

## 5. LOGO

### O símbolo — V

O símbolo do Vertex é um **V minimalista** construído com geometria precisa. Representa:
- o vértice (ponto mais alto de um triângulo ou caminho)
- direção, crescimento, convergência
- clareza e precisão formal

**Princípios do símbolo:**
- Traços retos, sem serifas ou curvas desnecessárias
- Peso visual equilibrado
- Funciona em qualquer tamanho, do favicon ao outdoor

### Versões do Logo

#### Versão Completa (principal)
```
[V] Vertex | OS
          OPERATING SYSTEM
```
Uso: cabeçalho da aplicação, apresentações, documentos oficiais.

#### Versão Compacta
```
[V] Vertex OS
```
Uso: notificações, PDFs, contextos com espaço reduzido.

#### Versão Ícone
```
[V]
```
Uso: favicon, ícone de aplicativo, avatar, marca d'água.

#### Versão Invertida
Logo branco sobre fundo preto. Usar quando o fundo é escuro.

#### Versão Monocromática
Logo em preto puro ou branco puro. Nunca usar variações de cinza intermediário.

### Tipografia do Logo

- Logotipo: **Inter Bold** — "VERTEX"
- Separador: "|" em peso regular, opacidade reduzida
- Produto: "OS" em Inter Bold
- Tagline: "OPERATING SYSTEM" em Inter Regular, letter-spacing: 0.2em, caixa alta

### Espaço de Proteção

O logo exige uma área de proteção equivalente à **altura da letra "V"** em todos os lados. Nenhum elemento externo deve entrar nessa área.

```
         ← X →
    ↑
    X   [  V  VERTEX | OS  ]   X
    ↓
         ← X →
```
*Onde X = altura do símbolo V*

### Tamanho Mínimo

| Contexto | Tamanho mínimo |
|---|---|
| Digital (logo completo) | 120px de largura |
| Digital (ícone) | 24px |
| Impresso (logo completo) | 30mm de largura |
| Impresso (ícone) | 8mm |

### Aplicação em UI

No produto, o logo aparece no **cabeçalho da sidebar**:
- Fundo: preto (#000000)
- Símbolo V: branco
- Texto "Vertex": branco
- Separador "|": branco, opacidade 30%
- "OS": branco
- "OPERATING SYSTEM": branco, tracking expandido, fonte menor

## 7. TIPOGRAFIA

### Fonte Principal

**Inter** — Google Fonts / variável

Inter foi escolhida por ser:
- Altamente legível em qualquer tamanho
- Otimizada para telas
- Disponível em todos os pesos necessários
- Amplamente suportada

### Escala Tipográfica

| Token | Tamanho | Peso | Line Height | Uso |
|---|---|---|---|---|
| `display` | 36px | 800 | 1.2 | Títulos de páginas de marketing |
| `h1` | 30px | 700 | 1.2 | Títulos principais de página |
| `h2` | 24px | 700 | 1.3 | Títulos de seção |
| `h3` | 20px | 600 | 1.4 | Subtítulos, cards principais |
| `h4` | 16px | 600 | 1.4 | Títulos de cards, grupos |
| `body-lg` | 16px | 400 | 1.6 | Texto corpo principal |
| `body` | 14px | 400 | 1.6 | Texto padrão da UI |
| `body-sm` | 13px | 400 | 1.5 | Descrições, metadados |
| `label` | 12px | 600 | 1.4 | Labels, badges, tags |
| `caption` | 11px | 400 | 1.4 | Anotações, timestamps |
| `micro` | 10px | 600 | 1.2 | Badges ultracompactos |

### Regras de Uso

**Títulos de página:**
- Fonte: Inter Bold (700)
- Sempre em `#0F172A` no modo light
- Não usar caixa alta em títulos longos

**Labels de seção (acima de títulos):**
- Fonte: Inter SemiBold (600)
- Texto em caixa alta
- Letter-spacing: 0.08em
- Cor: `#94A3B8` (cinza médio)
- Exemplo: "CONHECIMENTO", "FINANCEIRO"

**Texto de corpo:**
- Fonte: Inter Regular (400)
- Tamanho mínimo: 13px
- Nunca usar itálico para destaque — use peso ou cor

**Badges e Tags:**
- Fonte: Inter Bold (700) ou SemiBold (600)
- Tamanho: 10–12px
- Letter-spacing positivo quando em caixa alta

**Números e métricas:**
- Fonte: Inter Bold (700)
- Números grandes: 24–36px
- Nunca misturar pesos em uma mesma métrica

## 9. PADRÃO DE UI / UX

### Princípios

1. **Minimalismo** — Cada elemento presente deve ter um propósito. Se pode ser removido sem perda de informação, remova.
2. **Legibilidade** — A hierarquia visual deve guiar o olho sem esforço consciente.
3. **Consistência** — O mesmo padrão sempre. Nunca reinventar padrões já estabelecidos dentro do produto.
4. **Ação clara** — O usuário deve sempre saber o que pode fazer a seguir. CTAs visíveis, estados de loading explícitos.

### Sidebar

- Background: `#000000`
- Logo no topo, ocupando toda a largura com padding de 16px
- Grupos de módulos colapsáveis com chevron
- Item ativo: destaque com leve background e cor do módulo
- Subitens: indentados 12px, fonte 13px
- Hover: `rgba(255,255,255,0.07)`
- Largura compacta: apenas ícones, sem texto
- Transição de colapso: 200ms ease

### Dashboard / Página Principal

- Header da página com breadcrumb de módulo (ex: "FINANCEIRO") + título em H1
- Botão de ação principal (CTA) alinhado à direita do header
- Linha de stats/métricas logo abaixo do header: 3–4 cards em grid
- Conteúdo principal abaixo dos stats
- Espaço entre seções: 32px

### Cards

**Card padrão:**
- Background: `#FFFFFF`
- Border: `1px solid #E2E8F0`
- Border-radius: 16px
- Padding: 16–20px
- Shadow: `0 1px 3px rgba(0,0,0,0.06)`
- Hover shadow: `0 4px 12px rgba(0,0,0,0.08)`
- Transição: `box-shadow 150ms ease`

**Card de stat (métrica):**
- Ícone à esquerda em container colorido (cor do módulo, opacidade 10%)
- Número grande (24–28px, Bold) à direita
- Label abaixo do número (12px, Regular, cinza)

**Card de item (lista):**
- Layout horizontal: ícone/imagem | info principal | ações à direita
- Hover revela botões de ação (edit, delete) que ficam ocultos por padrão
- Chevron ou ícone de navegação no lado direito

### Modais

- Overlay: `rgba(0,0,0,0.5)`
- Modal: fundo branco, border-radius 20px, shadow grande
- Header com título + botão X no canto
- Footer com botões "Cancelar" (ghost) e CTA principal
- Largura máxima: 480px (formulários), 640px (conteúdo rico)
- Padding interno: 24px

### Tabelas

- Header: fundo `#F8FAFC`, texto 12px SemiBold, caixa alta, cor cinza
- Linhas: divider sutil `#F1F5F9` entre linhas
- Hover em linha: `#F8FAFC`
- Sem borders verticais
- Ações por linha: ocultas por padrão, reveladas no hover

### Estados

| Estado | Tratamento |
|---|---|
| **Loading** | Spinner indigo no centro, ou skeleton com `animate-pulse` |
| **Vazio** | Ícone ilustrativo + título + subtexto + CTA de criação |
| **Erro** | Mensagem clara + sugestão de ação |
| **Sucesso** | Toast no canto superior direito, 3s |
| **Hover** | Mudança de sombra ou cor de border, nunca escala |
| **Focus** | Ring indigo `2px`, offset `2px` |
| **Disabled** | Opacidade 50%, cursor `not-allowed` |

## 12. TOM DE VOZ E LINGUAGEM

### Princípios

**Claro antes de elegante.** Uma frase simples é sempre melhor que uma elaborada.

**Direto ao ponto.** O usuário não lê, ele escaneia. Cada palavra deve ser necessária.

**Humano, não robótico.** Sem linguagem de sistema. Em vez de "Operação concluída com sucesso", use "Salvo!".

**Profissional, não formal em excesso.** Tutear o usuário. Frases curtas. Verbos no imperativo para CTAs.

### Padrões de Linguagem na UI

**CTAs:**
- ✅ "Adicionar livro", "Salvar alterações", "Criar meta"
- ❌ "Submeter", "Confirmar operação", "Processar"

**Mensagens de sucesso:**
- ✅ "Livro adicionado!", "Progresso salvo", "Removido com sucesso"
- ❌ "Registro inserido com sucesso no banco de dados"

**Estados vazios:**
- ✅ "Sua biblioteca está vazia. Comece adicionando o primeiro livro."
- ❌ "Nenhum registro encontrado."

**Erros:**
- ✅ "Não conseguimos salvar. Tente novamente."
- ❌ "Erro 500: Internal Server Error"

**Labels de formulário:**
- Conciso, sem "por favor" ou "informe o"
- ✅ "Título *", "Autor *", "Data de início"
- ❌ "Por favor, informe o título do livro"

**Placeholders:**
- Exemplos reais e úteis
- ✅ `Atomic Habits`, `James Clear`, `https://...`
- ❌ `Digite aqui`, `Texto`, `Campo obrigatório`

### Idioma

Todo o produto é em **Português Brasileiro**. Sem mistura de idiomas na UI. Nomes técnicos de programação não aparecem para o usuário.

## 13. REGRAS DE USO

### O que PODE

- Usar o logo em fundo preto ou branco
- Usar a paleta de cores definida neste manual
- Adaptar o tom de voz às necessidades do módulo sem perder a personalidade
- Usar variações de tamanho do logo dentro dos limites mínimos
- Aplicar a cor do módulo como acento em ícones e badges

### O que NÃO PODE

- Distorcer, rotacionar ou aplicar efeitos ao logo
- Usar o logo em fundos coloridos que comprometam a legibilidade
- Criar submarcas ou variações do logo para módulos
- Usar fontes diferentes de Inter no produto
- Usar gradientes ou efeitos visuais não previstos neste manual
- Misturar bibliotecas de ícones diferentes
- Usar cores fora da paleta sem aprovação de design
- Usar texto em caixa alta para blocos de texto (só para labels)
- Aplicar sombras coloridas em elementos de UI
- Usar mais de uma cor de destaque por tela

### Aplicação em UI — Checklist de Conformidade

Antes de entregar qualquer tela, verificar:

- [ ] Fonte correta (Inter)
- [ ] Tamanhos dentro da escala tipográfica
- [ ] Espaçamentos em múltiplos de 8px
- [ ] Border-radius correto por tipo de componente
- [ ] Hierarquia visual clara (o que é mais importante é visualmente maior/mais forte)
- [ ] Estados definidos (hover, loading, vazio, erro)
- [ ] Ícones Lucide, tamanho correto
- [ ] Cores dentro da paleta
- [ ] Contraste adequado (WCAG AA)
- [ ] Labels em Português Brasileiro
- [ ] Placeholders com exemplos reais
