Quero evoluir a importação de faturas/extratos no módulo Financeiro para funcionar de forma profissional e adaptada ao contexto brasileiro.

OBJETIVO
Corrigir a lógica da competência da fatura, permitir seleção de cartão e melhorar o reconhecimento automático de padrões de bancos/cartões brasileiros.

IMPORTANTE
Não quero solução genérica.
Quero comportamento real de app financeiro brasileiro.

====================================================
1) COMPETÊNCIA DA FATURA (CORREÇÃO CRÍTICA)
====================================================

PROBLEMA
Hoje o sistema usa a data original da compra.

QUERO
Separar:

- data da compra (informativa)
- competência da fatura (financeira)

Exemplo:
Compra: 09/02/2026  
Fatura: Abril/2026  

Resultado:
→ lançamento deve entrar em ABRIL

IMPLEMENTAR:

- campo obrigatório: competência da fatura (mês/ano)
- salvar purchaseDate separadamente
- usar statementCompetence nos relatórios

====================================================
2) SELEÇÃO DE CARTÃO (OBRIGATÓRIO)
====================================================

Na tela de revisão:

Adicionar campo:

Cartão desta fatura

- dropdown com cartões cadastrados
- obrigatório antes de importar
- aplicado ao lote inteiro

====================================================
3) DETECÇÃO AUTOMÁTICA DO CARTÃO
====================================================

O sistema deve tentar identificar automaticamente o cartão com base no arquivo:

Exemplos:
- "Fatura Itaucard" → Itaú
- "Nubank" → Nubank
- "Latam Pass Black" → cartão Latam
- "Personnalité" → Itaú Personnalité

FUNCIONAMENTO:

- sugerir automaticamente o cartão
- mostrar como sugestão (não fixar)
- usuário confirma ou altera

====================================================
4) RECONHECIMENTO DE PADRÕES BRASILEIROS
====================================================

Melhorar parser para reconhecer formatos reais:

Itaú:
- padrão de tabela
- datas dd/mm/aaaa
- valores com vírgula

Nubank:
- descrições curtas
- valores diretos
- parcelamentos

Outros bancos:
- detectar padrões de:
  - data
  - valor
  - descrição

TRATAR:
- vírgula como decimal
- R$ no texto
- valores negativos
- textos mistos

====================================================
5) DETECÇÃO DE PARCELAMENTO
====================================================

Reconhecer automaticamente:

Exemplos:
- 1/3
- 2/12
- 3/6

Salvar:
- parcela atual
- total de parcelas

====================================================
6) TELA DE REVISÃO MELHORADA
====================================================

Adicionar no topo:

- cartão (auto detectado + editável)
- competência da fatura
- total de lançamentos

Cada linha:
- data (compra)
- descrição
- valor
- parcela
- categoria
- checkbox

====================================================
7) LÓGICA FINAL DE SALVAMENTO
====================================================

Salvar cada lançamento com:

- purchaseDate
- statementCompetence
- cardId
- valor
- descrição
- parcela

Relatórios devem usar:
→ statementCompetence

====================================================
8) EXPERIÊNCIA DO USUÁRIO
====================================================

Quero UX clara:

- deixar explícito:
  - qual cartão
  - qual mês da fatura
- evitar confusão entre data de compra e data financeira

====================================================
9) ENTREGAR
====================================================

Quero que você implemente:

1. competência da fatura separada
2. seleção obrigatória de cartão
3. detecção automática de cartão
4. parser adaptado para bancos brasileiros
